# 我把 8 页中国传统色文章，做成了一个能直接产出网站的技能



最近读到一篇题为「你绝对没见过的中国传统色」的文章，8 张图翻下来 5 种**双色搭配**：豆黄 × 群青、扶光 × 中国红、丹红 × 深海绿、水蓝 × 鹤蓝、淡粉 × 金红。配色意境极佳，但当你真要把这套色搬进一个网站，很快会撞到三堵墙：印刷 HEX 全部是错的（已用像素采样校正，例如豆黄印刷写 `#AE3B35`，实际是 `#FFDF70`），从「好看的色卡」到「可落地的 UI token」还差一大步，做出来的页面又几乎注定会落入"AI 模板脸"。

于是我做了一个叫 **website-ui-design** 的技能，把这篇文章沉淀成一个可重复使用的工作流。

## 一、5 种双色搭配，全部以 token 形式给出

| 搭配       | 暖色        | 冷色        | 适用场景       |
| -------- | --------- | --------- | ---------- |
| 豆黄 × 群青  | `#FFDF70` | `#153472` | 国潮活动、文化展   |
| 扶光 × 中国红 | `#EFC2A3` | `#DA0011` | 中式喜庆、女性品牌  |
| 丹红 × 深海绿 | `#FE5C1B` | `#1A3B32` | 茶饮、餐饮、潮牌东方 |
| 水蓝 × 鹤蓝  | `#9ED4F0` | `#144975` | 科技文化、博物馆   |
| 淡粉 × 金红  | `#EBD2D6` | `#E45600` | 茶器、手作、轻奢东方 |

每一组都同时给到 `--bg` / `--surface` / `--primary` / `--text` / `--accent` / `--border` 一套 CSS 变量，复制就能用。冷暖比例 6 : 4、墨色 `#161823` 与纸白 `#F2ECDE` 专做文字与留白——这些是反复试验后的搭配法则，不是巧合。

## 二、除了中国古典，还内置 7 种风格

新野兽派、极简瑞士、玻璃拟态、暗黑科技、复古孟菲斯、包豪斯、柔和有机，每种都自带 token、字体、组件语言。Brief 进来先做「Design Read」推断方向，再选风格；不再让 AI 凭训练数据均值乱选。

## 三、真正"去 AI 味"的强约束

这一项是这个技能的核心。它把 Leonxlnx/taste-skill 的去 AI 化方法论内嵌为强制收尾的 Pre-Flight Check，列明 16 条禁用项：三等分 feature 卡片、紫色 mesh 渐变、em-dash 滥用、scroll 提示、window.addEventListener('scroll') 黑魔法……任何一条没通过都不许交付。还有三个拨盘（VARIANCE / MOTION / DENSITY）量化控制版式方差与信息密度，确保不是千篇一律的居中套路。

## 四、组件级中国风落地

不仅是色卡，技能里直接提供带 5 个 pair 配色卡的语义化 HTML 模板——`assets/starter.html`。非对称 hero、zig-zag 特性区、CTA 底部对齐、双色搭配卡即开即用。

简言之：从一篇好看的文章，到一组可校验的 token，再到一个看起来"被人设计过"的网站——这就是这个技能要做的事。需要它的时候，对 WorkBuddy 说一句"用 website-ui-design 帮我设计 X"就行。
